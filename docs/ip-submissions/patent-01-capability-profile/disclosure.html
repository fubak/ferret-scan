Abstract

A computer-implemented system automatically generates structured security profiles for artificial intelligence command-line interface (AI CLI) agents by parsing their configuration files. The system classifies each declared permission into a canonical capability taxonomy (encompassing file access, code execution, shell access, network access, and related categories), assigns a risk level to each capability, and aggregates these into a holistic agent risk score with actionable remediation recommendations. The system supports multiple AI CLI platforms through a unified registry-based classification approach.

Technical Field

This invention relates to computer security, specifically to automated risk assessment of AI agent configurations by parsing and classifying capability declarations across heterogeneous AI CLI environments.

Background

AI CLI agents (including Claude Code, Cursor, Windsurf, Continue, Aider, and Cline) expose rich capability systems through configuration files. These capabilities — such as file write, shell execution, and network access — carry material security risks when granted inappropriately. Prior to this invention, no automated system existed to:

  • Parse configuration files from multiple AI CLI platforms through a unified interface
  • Map AI CLI-specific permission keys to a canonical, platform-independent capability taxonomy
  • Assign calibrated risk levels to individual capabilities
  • Generate a structured, actionable capability profile for security review

Existing security scanners treat AI CLI configs as generic configuration files, missing the semantic meaning of capability declarations. This gap leaves organizations unable to systematically audit what access they have granted to AI agents.

Detailed Description of the Invention

1. System Architecture

The capability profile generation system operates in three pipeline stages: Agent Detection, Capability Extraction, and Risk Assessment & Report Generation.

Figure 1. Three-stage pipeline: Agent Detection → Capability Extraction → Risk Assessment

2. AI CLI Registry (Stage 1)

The system maintains a static registry AI_CLI_CONFIGS mapping seven AI CLI platform identifiers to file path patterns and capability key mappings. Detection matches the input file path (filename and parent directory) against each platform's pattern list. The registry is extensible — adding support for a new AI CLI requires only a new registry entry.

3. Capability Taxonomy

The system defines eighteen canonical capability types covering the full surface area of AI agent permissions. Each type carries a base risk level used in aggregate scoring:

Figure 2. Eighteen-type capability taxonomy organized by risk level

4. Recursive Configuration Parsing

A key technical element is the findNestedValue() function, which recursively traverses any depth of JSON object to locate capability key declarations. This is necessary because AI CLI configurations place capability declarations at varying nesting depths. For JSON parsing failures, the system falls back to YAML-style regex key-value parsing, ensuring broad compatibility.

5. Tool-Name-to-Capability Mapping

The mapToolToCapability() function translates free-form tool name strings (e.g., "Bash", "WebFetch", "NotebookEdit") to canonical capability types. This allows the system to interpret allowedTools arrays without platform-specific parsing logic for each tool name variant.

Independent Claims

Claim 1 — System

A computer-implemented system for generating an AI agent capability profile, the system comprising: one or more processors configured to: receive a path to at least one configuration file associated with an artificial intelligence command-line interface (AI CLI) agent selected from a plurality of AI CLI agents; classify the AI CLI agent by matching the file path against a stored registry mapping file path patterns to AI CLI agent types; parse the configuration file to extract a set of declared permissions; for each extracted permission, map the permission to a canonical capability type selected from a predefined capability taxonomy comprising at least: file read, file write, code execution, shell access, and network access; assign a risk level from a predefined risk hierarchy to each canonical capability type; aggregate the risk levels to compute an overall agent risk score; and output a structured capability profile comprising the canonical capability types, their individual risk levels, and the overall agent risk score.

Claim 2 — Method

A computer-implemented method for classifying permissions in AI CLI agent configuration files, comprising: maintaining a registry mapping file path patterns to AI CLI platform identifiers and associated capability key sets; for a received configuration file, identifying a matching AI CLI platform identifier by comparing the file path against the registry; parsing the configuration file using a first parsing strategy comprising JSON deserialization with recursive key lookup; upon failure of the first parsing strategy, applying a second parsing strategy comprising pattern-based key-value extraction from the raw file content; for each capability key located by either strategy, mapping the key's value to a canonical capability type and an associated base risk level; computing an aggregate risk level by applying a monotonically escalating risk function over all mapped capability types having an allowed permission state; and emitting a structured report enumerating each capability type, its permission state, its risk level, and a set of generated remediation recommendations.

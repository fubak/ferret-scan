Abstract

A method and system detect coordinated, multi-file security attack patterns in AI CLI configuration deployments by constructing a file relationship graph using directory proximity and semantic naming relationships between AI CLI components, then searching for content pattern signatures that must span at least two related files. A quantitative correlation strength score combines pattern coverage, file participation, and proximity metrics. Detections produce structured findings identifying the attack pattern, involved files, attack vectors, and correlation confidence.

Technical Field

This invention relates to multi-file static analysis and security threat detection, specifically the detection of coordinated attack chains distributed across heterogeneous AI CLI configuration files through file relationship graph construction and cross-file pattern correlation.

Background

Sophisticated attackers distributing malicious content across multiple AI CLI configuration files (e.g., placing credential-stealing logic in a hook file while establishing network access in a skill file) evade single-file security scanners. Traditional cross-file taint analysis (used in conventional SAST) relies on program flow and data flow graphs between source code functions — techniques that do not apply to the configuration-file structure of AI CLI deployments.

The AI CLI environment introduces a novel relationship structure: files are related not by function call graphs but by (1) directory co-location and (2) semantic naming patterns specific to the AI CLI domain (hooks, skills, agents, settings, etc.). No prior system exploited these relationships for attack chain detection.

Detailed Description

1. File Relationship Graph Construction

The system builds a relationship graph over the set of discovered AI CLI configuration files using two independent edge-creation mechanisms:

  • Directory Distance Edges: A directory distance metric is computed as the number of path components in the relative path between two file directories. Files with distance ≤ 2 receive a relationship edge.
  • Semantic Naming Edges: File paths are tested against a library of AI CLI-specific naming relationship patterns. Any pair matching a pattern receives an edge regardless of directory distance. Example patterns: hooks↔skills, hooks↔agents, CLAUDE.md↔.mcp.json, settings.json↔config.*, agent↔skill, security/auth↔permission/access.
Figure 1. File relationship graph edges (directory proximity and semantic naming) with detected cross-file attack chain

2. Correlation Rule Structure

Correlation rules specify the conditions under which a cross-file pattern constitutes an attack chain. Each rule contains:

  • filePatterns: Substrings identifying which file paths are candidates for this rule
  • contentPatterns: Ordered list of regex patterns, each of which must match in at least one of the candidate files
  • description: Attack pattern description used to infer risk vectors

A rule fires when all content patterns are found across the set of files in a relationship cluster, with no requirement that any single file match more than one pattern.

4. Risk Vector Inference

When a correlation finding is emitted, the system infers attack risk vectors from the rule description text using keyword matching. Detected vectors include: Credential Exposure, Data Exfiltration, Privilege Escalation, Persistence Mechanism, Steganography/Hiding, Hook-Skill Chain, and Configuration Tampering. The final CorrelationFinding includes all inferred vectors in a riskVectors[] array alongside the numeric correlationStrength and relatedFiles[] list.

Independent Claims

Claim 1 — Method

A computer-implemented method for detecting multi-file security attack patterns in artificial intelligence command-line interface (AI CLI) configurations, the method comprising: discovering a plurality of configuration files associated with an AI CLI deployment; constructing a file relationship graph by: calculating a directory distance between each pair of configuration files; adding a relationship edge between pairs whose directory distance is within a threshold; adding relationship edges between pairs whose file names match predefined AI CLI semantic naming patterns indicating functional relationships between AI CLI components; for each edge-connected file cluster in the graph, searching for a set of content patterns defined by a correlation rule, wherein the content patterns must be present across at least two different files within the cluster; computing a correlation strength score based on a ratio of matched patterns to total patterns in the correlation rule, adjusted by the number of unique files matching at least one pattern; and generating a correlation finding when the correlation strength score exceeds a threshold, the finding identifying the matched files and the attack pattern represented by the correlation rule.

Claim 2 — System

A system for detecting coordinated attack chains in AI CLI configuration environments, comprising: one or more processors configured to execute: a relationship builder that constructs a graph of file relationships using both directory co-location proximity and semantic naming relationships specific to AI CLI component types including hooks, skills, agents, and configuration files; a pattern searcher that, for each correlation rule, reads file content across related files and locates regex pattern matches; a strength calculator that computes a numeric correlation strength from pattern coverage ratio, file participation ratio, and a proximity adjustment factor; a vector inferrer that derives attack risk vectors from rule description text by keyword analysis; and a finding emitter that produces structured correlation findings comprising related file paths, attack pattern description, inferred risk vectors, and correlation strength.
